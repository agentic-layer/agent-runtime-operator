= How-to Guides

Practical guides to solve specific problems and achieve particular goals with the Agent Runtime Operator.

== Installation

=== How to Install the Operator

==== Prerequisites

Before installing, ensure you have:

* Kubernetes cluster (v1.19+)
* kubectl configured to access your cluster
* cert-manager installed (for webhooks)

==== Install cert-manager

If cert-manager isn't already installed:

[source,bash]
----
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.18.2/cert-manager.yaml

# Wait for cert-manager pods to be ready
kubectl wait --for=condition=ready pod -l app.kubernetes.io/name=cert-manager -n cert-manager --timeout=300s
----

==== Installation Methods

===== Method 1: From Release Manifests

[source,bash]
----
# Install CRDs and operator from latest release
kubectl apply -f https://github.com/agentic-layer/agent-runtime-operator/releases/latest/download/install.yaml
----

===== Method 2: Using Kustomize

[source,bash]
----
# Clone the repository
git clone https://github.com/agentic-layer/agent-runtime-operator
cd agent-runtime-operator

# Install CRDs
make install

# Deploy using kustomize
kubectl apply -k config/default
----

===== Method 3: Development Installation

For development or customization:

[source,bash]
----
# Clone and build locally
git clone https://github.com/agentic-layer/agent-runtime-operator
cd agent-runtime-operator

# Install CRDs
make install

# Build and deploy
make docker-build docker-push IMG=<your-registry>/agent-runtime-operator:tag
make deploy IMG=<your-registry>/agent-runtime-operator:tag
----

==== Verify Installation

[source,bash]
----
# Check operator pods
kubectl get pods -n agent-runtime-operator-system

# Verify CRDs are installed
kubectl get crds | grep runtime.agentic-layer.ai

# Check webhook configuration
kubectl get validatingwebhookconfiguration | grep agent-runtime-operator
----

=== How to Uninstall the Operator

[source,bash]
----
# Delete all agent resources first
kubectl delete agents --all

# Remove the operator
make undeploy

# Remove CRDs (this will delete all agent resources)
make uninstall
----

== Configuration

=== How to Configure Different Agent Types

==== Google ADK Agents

[source,yaml]
----
apiVersion: runtime.agentic-layer.ai/v1alpha1
kind: Agent
metadata:
  name: adk-agent
spec:
  framework: google-adk
  image: my-registry/adk-agent:v1.0.0
  protocols:
    - type: A2A
  replicas: 2
----

=== How to Configure Environment Variables

[source,yaml]
----
apiVersion: runtime.agentic-layer.ai/v1alpha1
kind: Agent
metadata:
  name: configured-agent
spec:
  framework: google-adk
  image: my-registry/my-agent:latest
  protocols:
    - type: A2A
  env:
    - name: LOG_LEVEL
      value: "debug"
    - name: MODEL_ENDPOINT
      value: "https://api.openai.com/v1"
    - name: API_KEY
      valueFrom:
        secretKeyRef:
          name: agent-secrets
          key: api-key
----

=== How to Set Resource Limits

[source,yaml]
----
spec:
  resources:

      cpu: "500m"
      memory: "512Mi"
    requests:
      cpu: "100m" 
      memory: "128Mi"

=== How to Configure Node Placement

[source,yaml]
----
spec:
  nodeSelector:
    node-type: compute
  tolerations:
    - key: "agent-workload"
      operator: "Equal"
      value: "true"
      effect: "NoSchedule"
  affinity:
    podAntiAffinity:
      preferredDuringSchedulingIgnoredDuringExecution:
        - weight: 100
          podAffinityTerm:
            labelSelector:
              matchExpressions:
                - key: app
                  operator: In
                  values: ["my-agent"]
            topologyKey: kubernetes.io/hostname
----

== Scaling and Management

=== How to Scale Agents

==== Manual Scaling

[source,bash]
----
# Scale to 5 replicas using kubectl patch
kubectl patch agent my-agent -p '{"spec":{"replicas":5}}'

# Or edit the agent resource directly
kubectl edit agent my-agent
----

==== Update Agent Image

[source,bash]
----
# Update to new image version
kubectl patch agent my-agent -p '{"spec":{"image":"my-registry/my-agent:v2.0.0"}}'
----

=== How to Fix Common Problems

==== Image Pull Errors

Problem: `ErrImagePull` or `ImagePullBackOff`

Solutions:
1. Verify image exists and tag is correct
2. Check registry credentials
3. Ensure image registry is accessible from cluster

[source,bash]
----
# Check image pull secrets
kubectl get secrets
kubectl describe secret <image-pull-secret>

# Test image pull manually
docker pull <image-name>
----

==== Resource Constraints

Problem: Pod stuck in `Pending` state

Solutions:
1. Check node resources: `kubectl describe nodes`
2. Reduce resource requests in agent spec
3. Add more nodes or use smaller resource limits

[source,bash]
----
# Check node capacity
kubectl describe nodes | grep -A 5 "Allocated resources"

# Check resource quotas
kubectl describe quota
----

==== Webhook Certificate Issues

Problem: Webhook admission failures

Solutions:
1. Verify cert-manager is running
2. Check certificate status
3. Restart operator pods

[source,bash]
----
# Check cert-manager
kubectl get pods -n cert-manager

# Check certificates
kubectl get certificates -A

# Restart operator
kubectl rollout restart deployment -n agent-runtime-operator-system agent-runtime-operator-controller-manager
----

=== How to Extend Framework Support

To add support for a new agent framework:

1. **Update the CRD** to include the new framework in validation
2. **Extend the controller** to handle framework-specific logic
3. **Add tests** for the new framework
4. **Update documentation** with examples

[source,go]
----
// In agent_types.go, add to framework validation
// +kubebuilder:validation:Enum=google-adk;flokk;autogen;my-new-framework

// In agent_controller.go, add framework handling
switch agent.Spec.Framework {
case "my-new-framework":
    // Add framework-specific deployment logic
}
----
