= Contributing

We welcome contributions to the Agent Runtime Operator! Please follow these guidelines to ensure a smooth contribution process.

== Setup for Contributors

=== 1. Fork and Clone the Repository

[source,bash]
----
git clone https://github.com/YOUR_USERNAME/agent-runtime-operator
cd agent-runtime-operator
----

=== 2. Install Pre-commit Hooks

Pre-commit hooks are mandatory for all contributors:

[source,bash]
----
brew bundle
----

[source,bash]
----
# Install hooks for this repository
pre-commit install
----

=== 3. Verify Your Development Environment

[source,bash]
----
# Run all checks that pre-commit will run
make fmt vet lint test
----

== Code Style and Standards

=== Go Style

* Follow standard Go conventions and use `gofmt` for formatting
* Use meaningful variable and function names
* Add proper documentation for exported functions

=== Linting

Code must pass golangci-lint checks. Configuration is in `.golangci.yml`.

=== Testing

* All new features must include appropriate unit tests
* Maintain test coverage above 80%
* Write integration tests for complex functionality

=== Documentation

* Update relevant documentation for new features
* Use clear, concise language
* Include code examples where appropriate

== Development Workflow

=== 1. Create a Feature Branch

Create a branch from `main`:

[source,bash]
----
git checkout -b feature/PAAL-1234-your-feature-name
----

=== 2. Make Your Changes

Follow the code style guidelines and ensure your changes are well-tested.

=== 3. Run Development Checks

Before committing, run all development checks:

[source,bash]
----
# Format code
make fmt

# Run static analysis
make vet

# Run linting  
make lint

# Run unit tests
make test

# Generate updated manifests if needed
make manifests generate
----

=== 4. Test Your Changes

Run comprehensive tests to ensure everything works:

[source,bash]
----
# Run E2E tests to ensure everything works
make test-e2e
----

=== 5. Commit Your Changes

Commit your changes with a descriptive commit message:

[source,bash]
----
git add .
git commit -m "feat: add new agent protocol support

- Add support for HTTP protocol in Agent CRD
- Update controller to handle HTTP protocol configuration
- Add unit tests for HTTP protocol validation"
----

=== 6. Submit a Pull Request

Submit a pull request with:

* Clear description of the changes
* Reference to any related issues  
* Screenshots/logs if applicable
* Updated documentation if needed

== Contribution Types

=== Bug Fixes

* Include reproduction steps in the issue description
* Add regression tests to prevent future occurrences
* Update documentation if the bug was related to incorrect docs

=== New Features

* Discuss the feature in an issue before implementing
* Ensure backward compatibility
* Add comprehensive tests and documentation

=== Documentation Improvements

* Fix typos and improve clarity
* Add missing examples
* Update outdated information

== Code Review Process

1. **Automated Checks**: All PRs must pass automated tests and linting
2. **Peer Review**: At least one maintainer must review and approve the PR
3. **Testing**: New features must include appropriate tests
4. **Documentation**: Changes must include relevant documentation updates

== Getting Help

* **Issues**: Report bugs or request features via GitHub issues
* **Discussions**: Use GitHub discussions for questions and general discussion
* **Slack**: Join our community Slack channel (link in repository)

== Recognition

Contributors will be recognized in:

* Repository contributor list
* Release notes for significant contributions
* Annual contributor highlights

Thank you for contributing to the Agent Runtime Operator!
