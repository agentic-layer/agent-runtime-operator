= Configuration

The Agent Runtime Operator can be configured through environment variables and custom resources.

== Overview

Configuration is handled at two levels:

* **Operator Configuration**: Environment variables that control the operator behavior
* **Agent Configuration**: Custom resources that define individual agents

== Environment Variables

The operator supports the following environment variables:

[cols="2,1,3"]
|===
|Variable |Default |Description

|`ENABLE_WEBHOOKS`
|`true`
|Set to `false` to disable admission webhooks

|`METRICS_BIND_ADDRESS`
|`:8443`
|Address for metrics server

|`HEALTH_PROBE_BIND_ADDRESS`
|`:8081`
|Address for health probes
|===

== Setting Environment Variables

Environment variables can be set in the operator deployment:

[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: agent-runtime-operator-controller-manager
spec:
  template:
    spec:
      containers:
      - name: manager
        env:
        - name: ENABLE_WEBHOOKS
          value: "false"
        - name: METRICS_BIND_ADDRESS
          value: ":9090"
----
