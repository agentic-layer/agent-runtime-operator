= Custom Resources

The Agent Runtime Operator uses Custom Resource Definitions (CRDs) to manage agents. The primary resource is the `Agent` resource.

== Agent Resource

To deploy an agent, you define an `Agent` resource. Here's the structure:

[source,yaml]
----
apiVersion: runtime.agentic-layer.ai/v1alpha1
kind: Agent
metadata:
  labels:
    app.kubernetes.io/name: agent-runtime-operator
    app.kubernetes.io/managed-by: kustomize
  name: weather-agent
spec:
  framework: google-adk  # Supported: google-adk, flokk, autogen
  image: eu.gcr.io/agentic-layer/weather-agent:0.1.2
  protocols:
    - type: A2A  # Agent-to-Agent protocol
  replicas: 1  # Number of agent replicas (optional, default: 1)
----

== Specification Fields

=== Required Fields

[cols="1,1,3"]
|===
|Field |Type |Description

|`framework`
|string
|The agent framework to use. Supported values: `google-adk`, `flokk`, `autogen`

|`image`
|string
|The container image for the agent

|`protocols`
|array
|List of protocol configurations for the agent
|===

=== Optional Fields

[cols="1,1,2,1"]
|===
|Field |Type |Description |Default

|`replicas`
|integer
|Number of agent replicas to run
|1
|===

== Protocol Configuration

The `protocols` field defines how agents communicate:

[source,yaml]
----
protocols:
  - type: A2A  # Agent-to-Agent protocol
----

Currently supported protocol types:

* `A2A`: Agent-to-Agent communication protocol

== Example Configurations

=== Simple Agent

[source,yaml]
----
apiVersion: runtime.agentic-layer.ai/v1alpha1
kind: Agent
metadata:
  name: simple-agent
spec:
  framework: google-adk
  image: my-registry/simple-agent:latest
  protocols:
    - type: A2A
----

=== Scaled Agent

[source,yaml]
----
apiVersion: runtime.agentic-layer.ai/v1alpha1
kind: Agent
metadata:
  name: scaled-agent
spec:
  framework: flokk
  image: my-registry/scaled-agent:v1.0.0
  protocols:
    - type: A2A
  replicas: 3
----

== Deployment

Apply your agent configuration:

[source,bash]
----
kubectl apply -f my-agent.yaml
----

== Status and Monitoring

Check the status of your agent:

[source,bash]
----
# Check agent status
kubectl get agents

# Get detailed agent information
kubectl describe agent my-agent

# Check created deployments
kubectl get deployments -l app.kubernetes.io/name=my-agent
----
